# 1992A - Only Pluses

**Enlace:** https://codeforces.com/problemset/problem/1992/A

## Resumen del problema
Se te dan tres números enteros `a`, `b` y `c`. Tienes que realizar exactamente 5 operaciones. En cada operación, puedes elegir uno de los tres números e incrementarlo en 1. El objetivo es maximizar el producto `a * b * c` después de realizar las 5 operaciones.

## Input
-   La primera línea contiene un solo entero `t` ($1 \le t \le 1000$), el número de casos de prueba.
-   Para cada caso de prueba:
    -   Una sola línea contiene tres enteros `a`, `b` y `c` ($1 \le a, b, c \le 10$).

## Output
Para cada caso de prueba, imprime un solo entero: el valor máximo posible del producto `a * b * c` después de 5 operaciones.

## Restricciones

| Variable     | Rango      | Notas                               |
| :----------- | :--------- | :---------------------------------- |
| t (casos)    | [1, 1000]  |                                     |
| a, b, c      | [1, 10]    | Números iniciales                   |

## Forma de resolverlo

### Enfoque Algorítmico (Estrategia Codiciosa)
Para maximizar el producto de tres números con una suma fija de incrementos, la estrategia óptima es hacer que los números sean lo más cercanos posible entre sí. Esto se logra incrementando repetidamente el número más pequeño. Si hay varios números con el mismo valor mínimo, incrementar cualquiera de ellos dará el mismo resultado óptimo para el producto.

La lógica es la siguiente:
1.  **Identificar el Mínimo:** En cada una de las 5 operaciones, se busca el número con el valor más bajo entre `a`, `b` y `c`.
2.  **Incrementar el Mínimo:** Una vez identificado el número más pequeño, se le suma 1. Esto ayuda a "equilibrar" los valores, acercándolos entre sí, lo cual es clave para maximizar su producto.
3.  **Repetir:** Este proceso se repite 5 veces.
4.  **Calcular Producto Final:** Después de las 5 operaciones, se calcula el producto de los tres números resultantes `a * b * c`.

Este enfoque es una estrategia codiciosa (greedy) porque en cada paso se toma la decisión localmente óptima (incrementar el número más pequeño) con la esperanza de llegar a una solución globalmente óptima. En este caso, esta estrategia sí conduce al resultado máximo.

## Algoritmo
1.  Leer el número de casos de prueba `t`.
2.  Para cada caso de prueba:
    a.  Leer los tres enteros `a`, `b` y `c`.
    b.  Realizar un bucle 5 veces (para las 5 operaciones):
        i.  Comparar `a`, `b` y `c` para encontrar el valor mínimo.
        ii. Incrementar en 1 el número que tenga el valor mínimo. Si hay empates (varios números son mínimos), el código incrementa el primero que encuentra en el orden `a`, `b`, `c`.
    c.  Calcular el producto `a * b * c`.
    d.  Imprimir el producto.

## Notas Adicionales
*   **Maximización del Producto:** Para un conjunto de números con una suma constante, su producto es maximizado cuando los números son lo más cercanos posible entre sí. Al incrementar el número más pequeño en cada paso, estamos reduciendo la diferencia entre los números, lo que nos acerca a la distribución óptima.
*   **Número Fijo de Operaciones:** El hecho de que sean exactamente 5 operaciones simplifica el problema, ya que no necesitamos decidir cuándo parar.
