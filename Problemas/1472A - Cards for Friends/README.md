# 1472A - Cards for Friends

## Resumen del problema
Policarpio tiene una hoja de papel rectangular de tamaño $w \times h$. Quiere cortar esta hoja en piezas más pequeñas para dárselas a sus $n$ amigos. Puede realizar los siguientes cortes:
- Si el ancho $w$ es par, puede cortar una pieza de $w \times h$ en dos piezas de $w/2 \times h$.
- Si el alto $h$ es par, puede cortar una pieza de $w \times h$ en dos piezas de $w \times h/2$.
Cada corte duplica el número de piezas a lo largo de una de las dimensiones. Policarpio puede realizar tantos cortes como desee. Se pregunta si es posible obtener al menos $n$ piezas de papel.

## Input
- La primera línea contiene un solo entero `t` ($1 \le t \le 10^4$), el número de casos de prueba.
- Para cada caso de prueba:
    - La única línea contiene tres enteros `w`, `h`, `n` ($1 \le w, h, n \le 10^9$) — el ancho de la hoja, el alto de la hoja y el número de amigos de Policarpio.

## Output
Para cada caso de prueba, imprime "YES" si es posible obtener al menos `n` piezas, y "NO" en caso contrario.

## Restricciones

| Variable     | Rango      | Notas                               |
| :----------- | :--------- | :---------------------------------- |
| t (casos)    | [1, $10^4$] |                                     |
| w (ancho)    | [1, $10^9$] |                                     |
| h (alto)     | [1, $10^9$] |                                     |
| n (amigos)   | [1, $10^9$] | Número de piezas requeridas         |

## Forma de resolverlo

### Enfoque Algorítmico (Máximo de Cortes)
El problema se centra en maximizar el número de piezas que se pueden obtener de una hoja de papel. Cada vez que una dimensión (ancho o alto) es par, podemos cortar la hoja por la mitad a lo largo de esa dimensión, duplicando el número de piezas existentes. Este proceso se puede repetir hasta que ambas dimensiones sean impares, momento en el que no se pueden realizar más cortes que dupliquen las piezas.

La estrategia es la siguiente:
1.  **Inicializar:** Comenzar con 1 pieza (la hoja original).
2.  **Cortar por Ancho:** Mientras el ancho de la hoja sea un número par, se puede cortar por la mitad. Cada vez que se realiza este corte, el número de piezas se duplica y el ancho se reduce a la mitad.
3.  **Cortar por Alto:** De manera similar, mientras el alto de la hoja sea un número par, se puede cortar por la mitad. Cada vez que se realiza este corte, el número de piezas se duplica y el alto se reduce a la mitad.
4.  **Verificar Suficiencia:** Una vez que no se pueden realizar más cortes (ambas dimensiones son impares), se compara el número total de piezas obtenidas con el número de amigos (`n`). Si el número de piezas es mayor o igual a `n`, entonces es posible. De lo contrario, no lo es.

Este enfoque garantiza que se obtiene el número máximo posible de piezas, ya que cada corte que se puede hacer se realiza, y cada corte duplica el número de piezas.

## Algoritmo
1.  Leer el número de casos de prueba `t`.
2.  Para cada caso de prueba:
    a.  Leer `w` (ancho), `h` (alto) y `n` (número de amigos).
    b.  Inicializar una variable `postales` a 1 (representando la hoja original).
    c.  Mientras `w` sea par y `w > 0`:
        i.  Multiplicar `postales` por 2.
        ii. Dividir `w` por 2.
    d.  Mientras `h` sea par y `h > 0`:
        i.  Multiplicar `postales` por 2.
        ii. Dividir `h` por 2.
    e.  Si `postales` es mayor o igual a `n`, imprimir "YES".
    f.  De lo contrario, imprimir "NO".

## Notas Adicionales
*   Los bucles `while` se detienen cuando la dimensión se vuelve impar, ya que no se pueden realizar más cortes que dupliquen las piezas a lo largo de esa dimensión.
*   La condición `ancho > 0` y `alto > 0` en los bucles `while` es una precaución, aunque en este problema `w` y `h` siempre serán al menos 1.

| :------- | :--------------------- | :---------------------------------- |

| Espacial | O(1)                   | Se utiliza una cantidad constante de memoria para almacenar las variables. |
